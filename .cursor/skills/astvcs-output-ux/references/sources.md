# Output UX sources

- [Command Line Interface Guidelines](https://clig.dev/): keep success output brief and hide creator-only internals behind explicit detail or verbose modes.
- [NN/g progressive disclosure](https://www.nngroup.com/articles/progressive-disclosure/): show frequent, task-critical information first and defer advanced material behind a clear control.
- [Git diff options](https://git-scm.com/docs/diff-options): provide compact human summaries while retaining explicit detailed forms.
- [GitHub Files Changed, January 2026](https://github.blog/changelog/2026-01-22-improved-pull-request-files-changed-page-on-by-default/): prioritize keyboard navigation, landmarks, large-diff performance, and focused review.
- [Gerrit Review UI](https://gerrit-review.googlesource.com/Documentation/user-review-ui.html): support complete keyboard navigation and a discoverable `?` shortcut surface.
- [VS Code diff editor](https://code.visualstudio.com/docs/sourcecontrol/staging-commits): expose next and previous change actions and collapse unchanged regions.
- [VS Code accessibility skill](https://github.com/microsoft/vscode/blob/main/.github/skills/accessibility/SKILL.md): require keyboard operation, logical focus order, visible focus, ARIA labels and state, and dismissible help.
- [Anthropic frontend design skill](https://github.com/anthropics/claude-plugins-official/blob/main/plugins/frontend-design/skills/frontend-design/SKILL.md): make copy and visual decisions serve comprehension instead of decoration.
